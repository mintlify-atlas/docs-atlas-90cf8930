## Overview

My Auto Guide uses Supabase as its backend-as-a-service platform, providing:

- 🔐 **Authentication**: User registration and login
- 💾 **Database**: PostgreSQL database for storing vehicle data, maintenance records, and user information
- 📦 **Storage**: File storage for vehicle photos, documents (SOAT, Tecnomecánica, insurance, etc.)
- 🔄 **Real-time**: Live data synchronization

## Create a Supabase Project

<Steps>
  <Step title="Sign up for Supabase">
    Go to [supabase.com](https://supabase.com) and create a free account
  </Step>
  
  <Step title="Create a new project">
    Click **New Project** and fill in:
    - Project name (e.g., "my-auto-guide")
    - Database password (save this securely)
    - Region (choose closest to your users)
  </Step>
  
  <Step title="Wait for project setup">
    Supabase will provision your database and services (takes ~2 minutes)
  </Step>
  
  <Step title="Get your API credentials">
    Navigate to **Settings → API** and copy:
    - Project URL
    - Anon/Public Key (anon key)
  </Step>
</Steps>

## Database Setup

My Auto Guide requires specific database tables and schemas for storing vehicle data.

### Required Tables

Create the following tables in your Supabase project:

#### 1. Users and Profiles

```sql SQL Editor
-- User profiles (extends auth.users)
CREATE TABLE public.profiles (
  id UUID REFERENCES auth.users PRIMARY KEY,
  email TEXT UNIQUE,
  full_name TEXT,
  created_at TIMESTAMP WITH TIME ZONE DEFAULT NOW(),
  updated_at TIMESTAMP WITH TIME ZONE DEFAULT NOW()
);

-- Enable Row Level Security
ALTER TABLE public.profiles ENABLE ROW LEVEL SECURITY;

-- Policy: Users can read their own profile
CREATE POLICY "Users can read own profile"
  ON public.profiles FOR SELECT
  USING (auth.uid() = id);

-- Policy: Users can update their own profile
CREATE POLICY "Users can update own profile"
  ON public.profiles FOR UPDATE
  USING (auth.uid() = id);
```

#### 2. Vehicles

```sql SQL Editor
-- Vehicles table (cars and motorcycles)
CREATE TABLE public.vehicles (
  id UUID DEFAULT gen_random_uuid() PRIMARY KEY,
  user_id UUID REFERENCES auth.users NOT NULL,
  vehicle_type TEXT NOT NULL, -- 'car' or 'motorcycle'
  brand TEXT NOT NULL,
  model TEXT NOT NULL,
  year INTEGER,
  license_plate TEXT,
  current_mileage DECIMAL,
  vin TEXT,
  photo_url TEXT,
  created_at TIMESTAMP WITH TIME ZONE DEFAULT NOW(),
  updated_at TIMESTAMP WITH TIME ZONE DEFAULT NOW()
);

ALTER TABLE public.vehicles ENABLE ROW LEVEL SECURITY;

CREATE POLICY "Users can manage own vehicles"
  ON public.vehicles FOR ALL
  USING (auth.uid() = user_id);
```

#### 3. Maintenance Records

```sql SQL Editor
-- Maintenance records
CREATE TABLE public.maintenance_records (
  id UUID DEFAULT gen_random_uuid() PRIMARY KEY,
  vehicle_id UUID REFERENCES public.vehicles NOT NULL,
  maintenance_type TEXT NOT NULL, -- 'oil', 'filter', 'chain', 'soat', 'technomechanical'
  date DATE NOT NULL,
  mileage DECIMAL,
  next_due_date DATE,
  next_due_mileage DECIMAL,
  cost DECIMAL,
  notes TEXT,
  created_at TIMESTAMP WITH TIME ZONE DEFAULT NOW()
);

ALTER TABLE public.maintenance_records ENABLE ROW LEVEL SECURITY;

CREATE POLICY "Users can manage maintenance for own vehicles"
  ON public.maintenance_records FOR ALL
  USING (
    vehicle_id IN (
      SELECT id FROM public.vehicles WHERE user_id = auth.uid()
    )
  );
```

#### 4. Documents

```sql SQL Editor
-- Vehicle documents (SOAT, insurance, etc.)
CREATE TABLE public.vehicle_documents (
  id UUID DEFAULT gen_random_uuid() PRIMARY KEY,
  vehicle_id UUID REFERENCES public.vehicles NOT NULL,
  document_type TEXT NOT NULL, -- 'soat', 'technomechanical', 'insurance', 'property_card'
  file_url TEXT NOT NULL,
  expiry_date DATE,
  created_at TIMESTAMP WITH TIME ZONE DEFAULT NOW()
);

ALTER TABLE public.vehicle_documents ENABLE ROW LEVEL SECURITY;

CREATE POLICY "Users can manage documents for own vehicles"
  ON public.vehicle_documents FOR ALL
  USING (
    vehicle_id IN (
      SELECT id FROM public.vehicles WHERE user_id = auth.uid()
    )
  );
```

<Tip>
  Run these SQL commands in the Supabase **SQL Editor** (accessible from your project dashboard)
</Tip>

## Authentication Setup

My Auto Guide uses Supabase Auth for user authentication.

### Enable Email Authentication

<Steps>
  <Step title="Navigate to Authentication">
    In your Supabase dashboard, go to **Authentication → Providers**
  </Step>
  
  <Step title="Enable Email provider">
    Ensure **Email** is enabled (it's enabled by default)
  </Step>
  
  <Step title="Configure email templates (optional)">
    Customize confirmation and password reset email templates in **Authentication → Email Templates**
  </Step>
</Steps>

### Authentication Flow

The app implements:

1. **Registration**: New users create accounts with email/password
2. **Login**: Existing users authenticate with credentials
3. **Password Reset**: Users can request password reset emails
4. **Session Management**: Automatic token refresh and persistence

Authentication is handled in `lib/login_screen.dart` and `lib/registro_screen.dart`.

## Storage Setup

My Auto Guide stores vehicle photos and documents in Supabase Storage.

### Create Storage Buckets

<Steps>
  <Step title="Navigate to Storage">
    Go to **Storage** in your Supabase dashboard
  </Step>
  
  <Step title="Create buckets">
    Create the following public or private buckets:
    
    - `vehicle-photos` (for vehicle images)
    - `vehicle-documents` (for SOAT, insurance, etc.)
  </Step>
  
  <Step title="Configure bucket policies">
    Set up Row Level Security policies for each bucket:
    
    ```sql
    -- Allow authenticated users to upload
    CREATE POLICY "Users can upload own files"
      ON storage.objects FOR INSERT
      WITH CHECK (auth.uid()::text = (storage.foldername(name))[1]);
    
    -- Allow users to read own files
    CREATE POLICY "Users can read own files"
      ON storage.objects FOR SELECT
      USING (auth.uid()::text = (storage.foldername(name))[1]);
    
    -- Allow users to delete own files
    CREATE POLICY "Users can delete own files"
      ON storage.objects FOR DELETE
      USING (auth.uid()::text = (storage.foldername(name))[1]);
    ```
  </Step>
</Steps>

### File Upload Structure

Organize files by user ID:

```
vehicle-photos/
  ├─ {user_id}/
  │   ├─ {vehicle_id}_main.jpg
  │   └─ {vehicle_id}_gallery_1.jpg
  
vehicle-documents/
  ├─ {user_id}/
  │   ├─ {vehicle_id}_soat.pdf
  │   └─ {vehicle_id}_technomechanical.pdf
```

## Test Your Configuration

After setting up Supabase, test the connection:

```dart lib/main.dart
// Supabase client is globally accessible
final supabase = Supabase.instance.client;

// Test authentication
try {
  final response = await supabase.auth.signUp(
    email: 'test@example.com',
    password: 'testpassword123',
  );
  print('User created: ${response.user?.id}');
} catch (e) {
  print('Error: $e');
}
```

## Environment Variables (Optional)

For better security, consider using environment variables instead of hardcoding credentials.

Create a `.env` file:

```bash .env
SUPABASE_URL=https://your-project-ref.supabase.co
SUPABASE_ANON_KEY=your-anon-key-here
```

<Warning>
  Add `.env` to your `.gitignore` to prevent committing sensitive credentials
</Warning>
